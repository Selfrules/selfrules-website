# Glossario IT — selfrules.org

> Regola guida: se un termine tecnico è comprensibile a un HM italiano senza traduzione, resta in inglese. Se ha un equivalente naturale in italiano che non suona forzato, si usa l'italiano. Mai mescolare nella stessa frase.

## Termini con equivalente italiano (usare SEMPRE l'italiano nella prosa)

| EN | IT | Note |
|---|---|---|
| setup | **configurazione** | "setup" solo come metrica citata ("tempo di setup -35%") |
| framework | **schema** / **approccio** | "framework" solo come nome proprio (es. titolo "Build vs Buy Framework") |
| pattern | **schema ricorrente** / **approccio** | "pattern" solo in contesti strettamente tecnici |
| feature | **funzionalità** | Sempre. Nessuna eccezione |
| fix (verbo) / fixare | **risolvere** / **aggiustare** / **correggere** | "fixare" non esiste in italiano. Eliminare sempre |
| fix (sostantivo) | **correzione** | "fix" accettabile solo in contesto molto tecnico ("il fix era già lì") |
| revenue | **fatturato** | Sempre |
| owner | **responsabile** | Sempre |
| ranking | **classifica** / **ordine di priorità** | Secondo contesto |
| tool | **strumento** | "tool" solo nel nome proprio di un prodotto |
| scope | **ambito** / **perimetro** | Sempre |
| deployment | **rilascio** / **messa in produzione** | Sempre |
| ship / shippare | **portare in produzione** / **rilasciare** / **consegnare** | "shippare" non esiste. "ship" inaccettabile in prosa italiana |
| pipeline | **coda** / **in lavorazione** | Secondo contesto |
| marketing copy | **testi promozionali** | Sempre |
| funnel | **percorso** / **processo** | Secondo contesto |

## Regole di applicazione

1. Se il termine è il nome di un prodotto, un titolo, o una metrica citata, resta in inglese
2. Se il termine appare nella prosa narrativa, usa la versione italiana
3. Mai più di 2 anglicismi nella stessa frase. Se ne servono di più, riformula
4. I termini "accettati" si trattano come parole italiane (niente corsivo, niente virgolette)

## Registro

Il tono target per tutto il sito è: diretto, conversazionale, con "tu" implicito. Mai accademico, mai da paper. Scrivere come se stessi spiegando a un collega al caffè, non come se stessi presentando a una conferenza. Le sezioni che funzionano meglio oggi (hero, beliefs, Outside Work) sono il benchmark.
