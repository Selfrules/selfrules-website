# Phase 3 — UI Design Contract

> Visual and interaction contract for the UI Primitives phase: Button, Tag, Section, MetricCard, CaseStudyCard, ProjectCard, VerticalTimeline, TimelineNode, BlinkingCursor, TerminalPrompt, CountUpNumber, and /dev/components verification route. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (pure Tailwind v4 utilities) |
| Icon library | none (arrow in CTA buttons is a text character or inline SVG) |
| Font | Inter (variable, 400-700) + Space Grotesk (variable, 500-700) + JetBrains Mono (variable, 400-700) via next/font/google |

**Inherited from Phase 1.** No component library, no shadcn. All styling via Tailwind CSS v4 `@theme` tokens in globals.css. Locked project-level decision. Space Grotesk added in DESIGN-UPDATE-v25 for headings, titles, and metric numbers.

## Spacing Scale

Inherited from Phase 1 UI-SPEC. All values are multiples of 4.

| Token | Value | Usage in Phase 3 |
|-------|-------|-------------------|
| xs | 4px | Tag inline padding (py), icon-to-text gap in Button |
| sm | 8px | Tag horizontal padding (px), gap between credential tags |
| md | 16px | MetricCard internal gap between number/label/context, ProjectCard internal gaps |
| lg | 24px | Card internal padding mobile (minimum) |
| xl | 32px | MetricCard padding, CaseStudyCard padding mobile (p-8), ProjectCard padding |
| 2xl | 48px | CaseStudyCard padding desktop (md:p-12) |
| 3xl | 64px | Not used directly in component internals |

**Phase-specific spacing (inherited from Phase 1):**

| Token | Value | Usage in Phase 3 |
|-------|-------|-------------------|
| section-gap | `clamp(80px, 10vw, 160px)` | Section wrapper vertical padding (py) |
| content-width | 720px | Section wrapper default max-width |
| wide-width | 1080px | Section wrapper `wide` variant max-width |
| page-padding | `clamp(20px, 5vw, 80px)` | Section wrapper horizontal padding (px) |

**Exceptions:**
- Button height: 48px (h-12) -- standard touch target, not a spacing token
- MetricCard number font size uses `clamp()` which may produce non-4px-multiple values at intermediate viewport widths. This is acceptable for fluid typography.

## Color

**60/30/10 split:** 60% dominant background (#0A0A0B), 30% surface/cards/borders (#111113 + #1A1A1F), 10% accent (#E8A838) reserved for CTAs, metric numbers, and hover highlights.

Inherited from Phase 1. Phase 3 uses all tokens:

| Role | Value | Tailwind Token | Phase 3 Usage |
|------|-------|----------------|---------------|
| Dominant (60%) | `#0A0A0B` | `bg-primary` | MetricCard cell background, page background behind components |
| Surface | `#111113` | `bg-surface` | Tag background (optional border variant), TimelineNode card background |
| Hover surface | `#1A1A1F` | `bg-hover` | Not used directly (hover is expressed through border color, not bg) |
| Border default | `#1A1A1F` | `border-default` | All card borders (1px), MetricCard grid gap color, Tag border, VerticalTimeline line color (2px) |
| Text primary | `#F5F5F0` | `text-primary` | Card titles, metric labels, button secondary text |
| Text secondary | `#8A8A8E` | `text-secondary` | Card descriptions, tags, section labels, button secondary hover text |
| Text tertiary | `#5A5A5E` | `text-tertiary` | MetricCard context, TimelineNode dates, TerminalPrompt text |
| Accent (10%) | `#E8A838` | `accent` | See reserved-for list below |
| Accent hover | `#D4962F` | `accent-hover` | Button primary hover background |
| Border accent | `#E8A838` | `border-accent` | Card hover border color (at varying opacity) |
| Dark text | `#0A0A0B` | `text-dark` | Button primary text (dark on accent bg) |

**Accent reserved for (Phase 3 scope):**
1. Button primary -- background color
2. MetricCard number -- text color (Space Grotesk)
3. CaseStudyCard metric number -- text color (right side, Space Grotesk)
4. Card hover states -- border-accent/40 (40% opacity) on MetricCard, CaseStudyCard, ProjectCard; border-accent/50 on TimelineNode
5. Section label -- NOT accent (it is text-secondary; accent is reserved for interactive/data elements)
6. BlinkingCursor -- block fill color and underscore text color (#E8A838)
7. TimelineNode active node -- filled square in accent (#E8A838)
8. VerticalTimeline overlap segment -- accent color for Designer/Developer overlap period

**No destructive color needed in Phase 3.** No destructive actions exist in the component library.

## Interaction Contracts

### Button Hover (UICM-01)

| Variant | Default State | Hover State | Transition |
|---------|---------------|-------------|------------|
| Primary | `bg-accent`, `text-dark` (#0A0A0B) | `bg-accent-hover`, `translate-y(-1px)` | `transition: all 150ms ease` |
| Secondary | `bg-transparent`, `border border-default`, `text-primary` | `border-accent`, `text-accent` | `transition: all 150ms ease` |

Both variants:
- Height: 48px (h-12)
- Padding: 12px vertical, 24px horizontal (inferred from `px-8` = 32px in Figma prototype -- use `px-8`)
- Corrected padding from prototype: `h-12 px-8` (48px height, 32px horizontal padding)
- Font weight: 400
- Display: `inline-flex items-center justify-center`
- Border radius: 0px
- When `href` is provided: renders as `<a>` (or Next.js `<Link>`). When absent: renders as `<button>`.
- Arrow suffix (" ->") is part of the label string passed via props, not generated by the component

### MetricCard Hover (UICM-02)

| Property | Default | Hover | Transition |
|----------|---------|-------|------------|
| Border color | `border-default` (#1A1A1F) | `border-accent/40` (#E8A838 at 40% opacity) | `transition: border-color 200ms ease` |

MetricCard does NOT have translate-y hover. Only border color changes.

### CaseStudyCard Hover (UICM-03)

| Property | Default | Hover | Transition |
|----------|---------|-------|------------|
| Border color | `border-default` (#1A1A1F) | `border-accent/40` (#E8A838 at 40% opacity) | `transition: all 200ms ease` |
| Transform | none | `translate-y(-2px)` | included in `all 200ms` |

### ProjectCard Hover (UICM-04)

| Property | Default | Hover | Transition |
|----------|---------|-------|------------|
| Border color | `border-default` (#1A1A1F) | `border-accent/40` (#E8A838 at 40% opacity) | `transition: border-color 200ms ease` |

ProjectCard does NOT have translate-y hover. Only border color changes.

### TimelineNode Hover (UICM-05)

| Property | Default | Hover | Transition |
|----------|---------|-------|------------|
| Border color | `border-default` (#1A1A1F) | `border-accent/50` (#E8A838 at 50% opacity) | `transition: border-color 200ms ease` |

Per Claude's discretion area in CONTEXT.md: TimelineNode content block gets hover border-accent/50 for consistency with other card types. No translate-y. The node square itself does not change on hover -- only the content block border.

### BlinkingCursor Animation

| Property | Value |
|----------|-------|
| Animation | `opacity: 0 ↔ 1` |
| Timing | `step-end`, 1s infinite |
| Reduced motion | Static (no blink), cursor remains visible at full opacity |

CSS-only animation, no JavaScript. Client Component only for the `'use client'` directive to enable CSS animation class toggling with `prefers-reduced-motion`.

### Component API Contracts

**Button:**
```
Props:
  variant: 'primary' | 'secondary'    -- required
  children: React.ReactNode            -- required (label text including arrow)
  href?: string                        -- if provided, renders Link/a; if absent, renders button
  className?: string                   -- optional override
  ...rest: ButtonHTMLAttributes | AnchorHTMLAttributes
```

**Tag:**
```
Props:
  children: React.ReactNode            -- required (label text)
  bordered?: boolean                   -- default true (1px border + surface bg); false = no border, no bg
  className?: string
```

**Section:**
```
Props:
  children: React.ReactNode            -- required
  wide?: boolean                       -- default false (720px); true = 1080px max-width
  className?: string
  id?: string                          -- for anchor linking (e.g., id="contact")
```

**SectionHeader:**
```
Props:
  label: string                        -- monospace uppercase label (e.g., "HOW I WORK")
  title: string                        -- section title (rendered in Space Grotesk 700)
  className?: string
```

**MetricCard:**
```
Props:
  number: string                       -- e.g., "99%+", "-25%", "116"
  label: string                        -- metric name
  context: string                      -- one-line explanation
  className?: string
```
Note: MetricCard renders the number in Space Grotesk 700. Pages can optionally wrap the number display with `CountUpNumber` for animation -- this is a page-level decision, not built into MetricCard itself.

**CaseStudyCard:**
```
Props:
  tag: string                          -- e.g., "B2B SaaS" (rendered via Tag component)
  title: string                        -- result-led title (rendered in Space Grotesk 700)
  preview: string                      -- 1-2 sentence description
  metric: string                       -- large accent number (e.g., "-25%", rendered in Space Grotesk 700)
  metricLabel?: string                 -- short label under metric (e.g., "post-release incidents")
  className?: string
```

**ProjectCard:**
```
Props:
  title: string                        -- rendered in Space Grotesk 700
  description: string                  -- one-liner
  techStack: string[]                  -- rendered as Tag components
  status: 'active' | 'shipped' | 'coming-soon'
  statusLabel: string                  -- display text for status (i18n)
  href?: string                        -- optional link to repo/demo
  className?: string
```

**BlinkingCursor:**
```
Props:
  type: 'block' | 'underscore'        -- required
  className?: string                   -- optional override

'block': small filled rectangle in accent (#E8A838), rendered inline
'underscore': '_' character in accent color, rendered inline
CSS animation: opacity 0↔1, step-end, 1s infinite
Respects prefers-reduced-motion: static (no blink), cursor visible at full opacity
Client Component ('use client'). ~15 lines of code. NO library.
```

**TerminalPrompt:**
```
Props:
  className?: string                   -- optional override

Renders:
  <span className="font-mono text-sm text-tertiary">mattia@selfrules ~ $</span>
  + <BlinkingCursor type="block" />

Server Component. Composes BlinkingCursor (client) inside server markup.
```

**CountUpNumber:**
```
Props:
  value: string                        -- e.g., "-25%", "116", "99%+"
  className?: string                   -- optional override

Behavior:
  - IntersectionObserver trigger at threshold 0.3
  - Duration: 800ms, easing: ease-out
  - Animates from 0 to numeric part, preserves prefix (e.g., "-") and suffix (e.g., "%", "%+", "+")
  - prefers-reduced-motion: no animation, number appears immediately at final value
  - Custom implementation with requestAnimationFrame. ~30 lines. NO library.
  - Client Component ('use client')
```

**VerticalTimeline:**
```
Props:
  children: React.ReactNode            -- required (TimelineNode children)
  className?: string                   -- optional override

Renders:
  Container with vertical line (2px, var(--border-default) #1A1A1F) on the left side.
  Children (TimelineNode components) are positioned along the line.
  Server Component.
```

**TimelineNode:**
```
Props:
  role: string                         -- job title (rendered in Space Grotesk 700)
  company: string                      -- company/context
  dates: string                        -- e.g., "2023 - oggi" (rendered in JetBrains Mono)
  description?: string                 -- 1-2 sentences (optional)
  isActive?: boolean                   -- default false; true = node filled accent
  className?: string                   -- optional override

Node visual:
  - Square shape: 8-10px, border-radius: 0 (consistent with 0px identity)
  - isActive=true: filled in accent (#E8A838)
  - isActive=false: outlined (1px border var(--border-default), transparent fill)
  - Horizontal connector: 1px line from node to content block on the right

Layout:
  - Line on left, node on line, horizontal connector, content block on right
  - Responsive: stays vertical on all viewports (no horizontal layout on desktop)

Designer/Developer overlap:
  - The segment of vertical line between overlapping career phases uses accent color (#E8A838)
    or displays an "overlap" badge. Implementation detail at page level.

Server Component.
```

## Layout Contracts

### MetricCard Grid (UICM-02, D-03)

The MetricCard grid uses the "shared border" pattern from the Figma prototype:

```
Container:
  display: grid
  grid-template-columns: repeat(3, 1fr)  -- desktop
  grid-template-columns: repeat(2, 1fr)  -- tablet (md breakpoint)
  grid-template-columns: repeat(1, 1fr)  -- mobile (default)
  gap: 1px
  background: var(--border-default)       -- the gap IS the border
  border: 1px solid var(--border-default)

Each MetricCard cell:
  background: var(--bg-primary)           -- fills the cell, gap shows through as border
  padding: 32px
  MetricCard number: font-heading (Space Grotesk), NOT font-mono
```

This creates the effect of shared 1px borders between cells (like a table grid). The container background color bleeds through the 1px gap.

### CaseStudyCard Layout (UICM-03, D-02)

```
Card:
  display: flex
  flex-direction: row (desktop) / column (mobile)
  border: 1px solid var(--border-default)
  padding: 32px mobile (p-8) / 48px desktop (md:p-12)
  gap: 24px (between left content and right metric)

Left side (flex: 1):
  Tag (top)
  Title (below tag, mt-3, font-heading Space Grotesk 700)
  Preview text (below title, mt-2)

Right side (flex-shrink: 0, self-center):
  Large accent number (clamp 40-72px, font-heading Space Grotesk 700, NOT font-mono)
  Optional metric label below (13px, text-tertiary)
  Arrow icon (inline SVG, 16px, accent color, to the right of number)
```

On mobile (<768px), the right-side metric moves below the preview text (flex-direction: column).

### ProjectCard Layout (UICM-04)

```
Card:
  border: 1px solid var(--border-default)
  padding: 32px (p-8)

Content stack (vertical):
  Row 1: Title (font-heading Space Grotesk 700) + status dot + status label (inline)
  Row 2: One-liner description (mt-2)
  Row 3: Tech stack tags (mt-4, flex wrap, gap-2)
  Row 4: Optional link (mt-4, accent color, arrow suffix)

Status dot:
  width: 8px, height: 8px, border-radius: 50% (EXCEPTION to 0px rule -- dots are circular)
  active: #E8A838 (accent)
  shipped: #4ADE80 (green)
  coming-soon: #8A8A8E (text-secondary)
```

**Exception to 0px border-radius:** Status dots use `border-radius: 50%` because they are circular indicators, not rectangular UI elements. This is the only exception.

### VerticalTimeline + TimelineNode Layout (UICM-05)

```
VerticalTimeline container:
  position: relative
  padding-left: 40-48px (space for line + node + connector + gap)

Vertical line:
  position: absolute
  left: 12-16px (centered on node position)
  top: 0, bottom: 0
  width: 2px
  background: var(--border-default) (#1A1A1F)

  Overlap segment (Designer/Developer period):
    Same line, but background: var(--accent) (#E8A838) for the overlapping range
    Or: an "overlap" badge positioned at the midpoint of the overlapping segment

TimelineNode:
  position: relative
  margin-bottom: 32px (gap between nodes)

  Node square:
    position: absolute
    left: -(offset to align with vertical line center)
    width: 8-10px, height: 8-10px
    border-radius: 0 (square -- consistent with 0px identity)
    isActive=true: background var(--accent) (#E8A838)
    isActive=false: border 1px solid var(--border-default), background transparent

  Horizontal connector:
    position: absolute
    from node right edge to content block left edge
    height: 1px
    background: var(--border-default)

  Content block (right of connector):
    border: 1px solid var(--border-default)
    background: var(--bg-surface) (#111113)
    padding: 24px (p-6)

    Row 1: Dates (font-mono, text-tertiary, 13px)

    Row 4: Description (body, text-secondary, mt-2) -- optional

Responsive:
  Stays vertical on all viewports. No horizontal layout.
  On mobile: padding-left may reduce slightly, content block takes remaining width.
```

## Accessibility Requirements (Phase 3 scope)

| Requirement | Implementation |
|-------------|----------------|
| A11Y-02 Focus styles | `outline: 2px solid #E8A838`, `outline-offset: 2px` on Button (both variants) |
| A11Y-04 Semantic HTML | Cards use `<div>` (not interactive, not `<article>` unless standalone); Section uses `<section>`; Button renders `<button>` or `<a>` based on `href` prop; VerticalTimeline uses `<ol>` or `<div>` with appropriate role |
| WCAG AA contrast | All text/bg combinations must pass 4.5:1. Key pairs: text-primary (#F5F5F0) on bg-primary (#0A0A0B) = 18.1:1 PASS; text-secondary (#8A8A8E) on bg-primary (#0A0A0B) = 5.3:1 PASS; text-tertiary (#5A5A5E) on bg-primary (#0A0A0B) = 3.2:1 BORDERLINE -- acceptable only for supplementary context text per WCAG AA large-text exception at 13px+ |
| Button a11y | When rendering as `<a>`: provide meaningful text content. When rendering as `<button>`: include `type="button"` default |
| Status dot | Status meaning conveyed via text label, not color alone (color-blind safe) |
| Reduced motion | BlinkingCursor: `prefers-reduced-motion: reduce` disables blink animation, cursor stays visible. CountUpNumber: `prefers-reduced-motion: reduce` disables count-up, number renders at final value immediately |
| Timeline nodes | Active/inactive state conveyed via both color AND position (last node = current role), not color alone |
