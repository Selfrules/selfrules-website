## Scroll Animation Contract

Source: 06-CONTEXT.md decisions D-01 through D-04.

### Animation: Section Fade-Up

| Property | Value |
|----------|-------|
| Name | `fade-up` |
| Applied to | Every `<section>` component except Hero |
| Initial state | `opacity: 0; transform: translateY(24px)` |
| Final state | `opacity: 1; transform: translateY(0)` |
| Duration | 600ms |
| Easing | `ease-out` |
| Trigger | IntersectionObserver, threshold 0.1 (10% visibility) |
| rootMargin | `0px 0px 0px 0px` (no offset -- trigger when 10% of section enters viewport) |
| Fires once | Yes -- `{ once: true }` on observer. No reverse animation on scroll up. |
| Hero section | NO animation. Hero content is visible immediately on page load. |
| Stagger | None. Each section animates independently when it enters viewport. No stagger on individual cards within a section. |

### CSS Implementation

```css
/* In globals.css */
@keyframes fade-up {
  from {
    opacity: 0;
    transform: translateY(24px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.animate-fade-up {
  animation: fade-up 600ms ease-out forwards;
}

.animate-fade-up-initial {
  opacity: 0;
  transform: translateY(24px);
}

@media (prefers-reduced-motion: reduce) {
  .animate-fade-up-initial {
    opacity: 1;
    transform: none;
  }
  .animate-fade-up {
    animation: none;
    opacity: 1;
    transform: none;
  }
}
```

### Animation: Timeline Node Illumination (Optional Polish)

Source: DESIGN-UPDATE-v25.md Change 7.

| Property | Value |
|----------|-------|
| Name | `timeline-node-illuminate` |
| Applied to | Timeline nodes (square, 8-10px) in the About page Career Path section |
| Default state | Node border 1px #1A1A1F, transparent fill |
| Active state | Node border 1px #E8A838, fill #E8A838 at 20% opacity |
| Last node | Always filled #E8A838 (current role -- no animation needed, starts in active state) |
| Trigger | IntersectionObserver on the corresponding content block, threshold 0.3 |
| Fires once | Yes -- `{ once: true }`. Node stays illuminated after activation. |
| Duration | 300ms transition on border-color and background-color |
| Easing | `ease-out` |
| Uses same pattern as | Section fade-up (IntersectionObserver, `once: true`) |

```css
.timeline-node {
  border: 1px solid var(--color-border-default);
  background: transparent;
  transition: border-color 300ms ease-out, background-color 300ms ease-out;
}

.timeline-node-active {
  border-color: var(--color-accent);
  background-color: rgba(232, 168, 56, 0.2);
}

.timeline-node-current {
  border-color: var(--color-accent);
  background-color: var(--color-accent);
}

@media (prefers-reduced-motion: reduce) {
  .timeline-node {
    transition: none;
  }
  /* Nodes show final illuminated state immediately */
  .timeline-node-active {
    border-color: var(--color-accent);
    background-color: rgba(232, 168, 56, 0.2);
  }
}
```

### Reduced Motion (A11Y-05)

| Condition | Behavior |
|-----------|----------|
| `prefers-reduced-motion: reduce` | All animations disabled. Content visible immediately with `opacity: 1; transform: none`. |
| `prefers-reduced-motion: no-preference` | Fade-up animations play as specified. |
| **BlinkingCursor** (Phase 3 component) | `prefers-reduced-motion: reduce`: cursor is STATIC -- no blinking. Cursor element remains visible but the `@keyframes blink` animation is disabled. The block cursor or underscore character renders at full opacity without oscillation. |
| **CountUpNumber** (Phase 3 component) | `prefers-reduced-motion: reduce`: number appears IMMEDIATELY at its final value. No count-up animation from 0. The component skips the `requestAnimationFrame` loop entirely and renders the target number on mount. |
| **Timeline node illumination** | `prefers-reduced-motion: reduce`: nodes show their final illuminated state immediately with no transition. CSS `transition: none` on `.timeline-node`. |

CSS rules for Phase 3 component reduced-motion behavior (to be applied in Phase 6 polish):

```css
/* BlinkingCursor reduced motion */
@media (prefers-reduced-motion: reduce) {
  .blinking-cursor {
    animation: none;
    opacity: 1;
  }
}

/* CountUpNumber: handled in JS -- check prefers-reduced-motion via
   window.matchMedia('(prefers-reduced-motion: reduce)').matches
   If true, set displayed value to target immediately, skip rAF loop. */
```

## Accessibility Contract (Phase 6 Additions)

| Requirement | ID | Specification |
|-------------|-----|---------------|
| Skip-to-content | A11Y-01 | Hidden link, visible on focus, targets `#main-content`. See Focus Styles section above. |
| Visible focus | A11Y-02 | 2px solid accent outline, 2px offset, on `:focus-visible` only. Applied globally. |
| Color contrast | A11Y-03 | Text primary #F5F5F0 on #0A0A0B = 17.4:1 (AAA). Text secondary #8A8A8E on #0A0A0B = 4.9:1 (AA pass). Accent #E8A838 on #0A0A0B = 8.6:1 (AAA). Verify #8A8A8E at 14px body -- if used at smaller sizes, bump to #9A9A9E or increase to 16px minimum. |
| Semantic HTML | A11Y-04 | `<header>` for navbar, `<nav>` for navigation, `<main id="main-content">` for page content, `<article>` for notes post, `<footer>` for footer. Each `<section>` has `aria-label` or heading. |
| Reduced motion | A11Y-05 | `prefers-reduced-motion: reduce` disables all CSS animations and transitions, sets `scroll-behavior: auto`. Additionally: BlinkingCursor renders static (no blink), CountUpNumber shows final value immediately (no count-up), timeline nodes show illuminated state with no transition. See Reduced Motion table in Scroll Animation Contract. |
