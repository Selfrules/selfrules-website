# Plan 06-01: SEO Metadata Infrastructure

## Objective

Add complete SEO metadata to every page: meta title, description, Open Graph tags, canonical URLs, hreflang alternates, JSON-LD structured data, sitemap.xml, robots.txt, and a static OG image. After this plan, every page is fully discoverable and correctly indexed by search engines in both languages.

<action>
Add `generateMetadata` export to each page file. Pattern for each:

```typescript
import type { Metadata } from 'next';
import { getTranslations } from 'next-intl/server';
import { createPageMetadata } from '@/lib/metadata';

type Props = { params: Promise<{ locale: string }> };

export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: 'metadata' });
  return createPageMetadata({
    locale,
    path: '/about',  // adjust per page
    title: t('about.title'),  // adjust per page
    description: t('about.description'),  // adjust per page
  });
}
```

Page-specific values:
- Homepage (`page.tsx`): path `'/'`, use `title: { absolute: t('home.title') }` to override template (homepage has its own full title). Set title.absolute in the return by spreading createPageMetadata result and overriding title.
- About: path `'/about'`, keys `about.title`, `about.description`
- Work: path `'/work'`, keys `work.title`, `work.description`
- Lab: path `'/lab'`, keys `lab.title`, `lab.description`
- Approach: path `'/approach'`, keys `approach.title`, `approach.description`
- Notes listing: path `'/notes'`, keys `notes.title`, `notes.description`
- Notes post: path `'/notes/why-i-prototype-in-code'`, keys `notesPost.title`, `notesPost.description`. Also add `type: 'article'` to openGraph.

IMPORTANT for homepage: The homepage title should NOT use the template pattern. Override with `title: { absolute: t('home.title') }` to get the full title without the " -- Mattia De Luca" suffix.

IMPORTANT: In Next.js 16, `params` is a Promise. Always `await params` before accessing locale.

<read_first>
- .planning/phases/06-seo-polish-deploy/06-RESEARCH.md (Pattern 3 and Pattern 6)

<action>
1. Create `src/app/sitemap.ts`:
   ```typescript
   import type { MetadataRoute } from 'next';

   const baseUrl = 'https://selfrules.org';

   const pages = ['', '/about', '/work', '/lab', '/approach', '/notes', '/notes/why-i-prototype-in-code'];

   export default function sitemap(): MetadataRoute.Sitemap {
     return pages.flatMap((path) => [
       {
         url: `${baseUrl}${path}`,
         lastModified: new Date(),
         alternates: {
           languages: {
             en: `${baseUrl}${path}`,
             it: `${baseUrl}/it${path}`,
           },
         },
       },
       {
         url: `${baseUrl}/it${path}`,
         lastModified: new Date(),
         alternates: {
           languages: {
             en: `${baseUrl}${path}`,
             it: `${baseUrl}/it${path}`,
           },
         },
       },
     ]);
   }
   ```

2. Create `src/app/robots.ts`:
   ```typescript
   import type { MetadataRoute } from 'next';

   export default function robots(): MetadataRoute.Robots {
     return {
       rules: {
         userAgent: '*',
         allow: '/',
       },
       sitemap: 'https://selfrules.org/sitemap.xml',
     };
   }
   ```

IMPORTANT: Both files go in `src/app/` (NOT inside `[locale]/`). The sitemap must list ALL URLs for all locales.
</action>
