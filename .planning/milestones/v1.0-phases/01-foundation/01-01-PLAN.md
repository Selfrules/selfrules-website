# Plan 01-01: Scaffold Next.js 16, Configure Design Tokens + Fonts + i18n Routing

> Foundation plan. Creates a working Next.js 16 project with Tailwind v4 design tokens, 3 custom fonts (Inter, Space Grotesk, JetBrains Mono), and next-intl i18n routing. Outputs a placeholder page that verifies all systems work.

## Context

This is the very first plan — no existing code. Everything is greenfield. The project scaffolds with `create-next-app@latest`, then configures: (1) Tailwind v4 CSS-first design tokens in globals.css, (2) three Google Fonts via next/font bridged into Tailwind with `@theme inline`, (3) next-intl i18n routing with proxy.ts (Next.js 16 convention), and (4) a placeholder page in both EN and IT that visually verifies tokens, fonts, and locale routing.

### Key decisions from CONTEXT.md

- **D-01:** Public GitHub repository
- **D-02:** i18n message keys grouped by namespace. Single JSON per locale with nested structure
- **D-03:** Token organization at Claude's discretion — idiomatic Tailwind v4 @theme approach
- **D-04:** Placeholder pages EN/IT with minimal verifiable content (headline, color swatches, font names, locale indicator)

### Critical pitfalls to avoid

- **Pitfall #1:** Use `proxy.ts` (NOT middleware.ts) — Next.js 16 breaking change
- **Pitfall #2:** Call `setRequestLocale(locale)` before any `useTranslations` in every layout/page + export `generateStaticParams`
- **Pitfall #3:** No `tailwind.config.ts` — Tailwind v4 is CSS-first, use `@theme` in globals.css only
- **Pitfall #4:** Set `localeDetection: false` to prevent cookie redirect loops
- **Pitfall #5:** Apply font variable classes on `<html>` (not `<body>`) for Tailwind v4 to resolve them
- **Pitfall #6:** `await params` — Next.js 16 params is a Promise
- **Pitfall #7:** Set all `--radius-*` tokens to 0px AND add global CSS reset `* { border-radius: 0 !important; }`
- **Pitfall #11:** Border radius leaking from browser defaults — global reset required
- **Pitfall #14:** params is a Promise in Next.js 16 — always `const { locale } = await params;`

### Fonts (from UI-SPEC + DESIGN-UPDATE-v25)

| Font | CSS Variable | Tailwind Token | Weights | Used For |
|------|-------------|----------------|---------|----------|
| Inter | `--font-inter` | `font-sans` | 400, 700 | Body text, small text |
| Space Grotesk | `--font-heading` | `font-heading` | 400, 700 | Display, headings, metric numbers |
| JetBrains Mono | `--font-jetbrains` | `font-mono` | 400, 700 | Tags, labels, monospace elements |

```typescript
import { setRequestLocale } from 'next-intl/server';
import { NextIntlClientProvider } from 'next-intl';
import { getMessages } from 'next-intl/server';
import { routing } from '@/i18n/routing';
import { notFound } from 'next/navigation';
import { hasLocale } from 'next-intl';

export function generateStaticParams() {
  return routing.locales.map((locale) => ({ locale }));
}

export default async function LocaleLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;

  if (!hasLocale(routing.locales, locale)) {
    notFound();
  }

  setRequestLocale(locale);
  const messages = await getMessages();

  return (
    <NextIntlClientProvider messages={messages}>
      {children}
    </NextIntlClientProvider>
  );
}
```

**Critical details:**
- `generateStaticParams` MUST be exported — required for static rendering of [locale] routes
- `setRequestLocale(locale)` MUST be called BEFORE any translation access — prevents dynamic rendering (Pitfall #2)
- `params` is `Promise<{ locale: string }>` — Next.js 16 breaking change (Pitfall #14)
- `await params` before using locale — Next.js 16 requirement
- `hasLocale` check + `notFound()` — prevents invalid locales from rendering
</actions>

## Success Criteria (Phase 1 Gate)

All four phase success criteria from ROADMAP.md must be TRUE before Phase 1 is complete:

1. **Locale routing works:** Visiting `/` shows an EN page, visiting `/it` shows an IT page
2. **Static rendering:** Running `next build` produces static output (circle icons, not lambda) for all routes
3. **Design tokens available:** Colors, spacing, typography scale, 0px border-radius are available as Tailwind utilities throughout the project
4. **Fonts render correctly:** Inter, Space Grotesk, and JetBrains Mono render correctly with no layout shift on page load

## Verification Commands

```bash
# Build succeeds
npm run build

# Check for static routes (should see ○, not λ)
npm run build 2>&1 | grep -E '○|λ'

# Verify next version
grep '"next":' package.json

# Verify next-intl installed
grep 'next-intl' package.json

# Verify no tailwind config file
ls tailwind.config.ts 2>/dev/null; echo "should be empty"

# Verify proxy.ts exists (not middleware.ts)
ls src/proxy.ts
ls src/middleware.ts 2>/dev/null; echo "should be empty"

# Verify design tokens
grep '#0A0A0B' src/app/globals.css
grep '#E8A838' src/app/globals.css
grep 'border-radius: 0' src/app/globals.css

# Verify fonts
grep 'Space_Grotesk' src/lib/fonts.ts
grep 'font-heading' src/app/globals.css

# Verify locale detection disabled
grep 'localeDetection: false' src/i18n/routing.ts

# Verify static rendering setup
grep 'setRequestLocale' src/app/[locale]/layout.tsx
grep 'generateStaticParams' src/app/[locale]/layout.tsx
```
