| Title | clamp(28px, 3vw, 36px) | 700 | 1.2 | Space Grotesk (`font-heading`) | Section titles ("Come lavoro", "Il percorso", "Numeri, non parole", "Cosa faccio ora", "Parliamo"), card titles (pillar headings, "Now" card headings, timeline role titles, email link in contact) |
| Body | 16px | 400 | 1.6 | Inter (`font-sans`) | Hero subtitle, "How I Work" intro paragraph, pillar descriptions, timeline block text, "Now" card descriptions, contact text. For emphasis within body (hero subtitle, intro paragraphs), increase line-height to 1.7 instead of changing size. **Signature phrase:** Inter 400 italic, 14-16px, text-secondary (#8A8A8E) — uses Body/Label size with italic modifier, not a separate role. |
| Label | 14px | 400 | 1.4 | JetBrains Mono (`font-mono`) | Section labels, credential tags, timeline dates, metric context lines, availability tag, terminal prompt text. Always `uppercase`, `letter-spacing: 0.05em`. Visual distinction from body achieved via font family (JetBrains Mono), uppercase transform, and letter-spacing -- not via weight. |

Additional typography rules:
- Bold text within body copy (e.g., **25%**, **99%+**): weight 700, same size as surrounding text
- Card titles use the title clamp at its natural responsive size; they do not need a separate fixed size
- Email address in contact section: title size, JetBrains Mono, accent color on hover
- Only two weights are used across the entire page: 400 (body, labels, signature) and 700 (headlines, titles, inline bold, metric numbers)
- Terminal prompt (`mattia@selfrules ~ $`): JetBrains Mono, 14px (label size), color #5A5A5E (text-tertiary), NOT uppercase (exception to label convention)
- Metric numbers use Space Grotesk 700 (NOT JetBrains Mono) -- metric context labels below numbers still use JetBrains Mono

## Layout Contract

### Hero Section (HOME-01) -- CRITICAL: 100vh ENFORCEMENT

**Height:** `min-h-screen` (100vh). The hero MUST fill the entire viewport. At first load, the user sees ONLY the hero. NO content from the Approach/HowIWork section below must be visible. This is a non-negotiable design requirement.

**Content positioning:** Content is vertically positioned in the UPPER portion of the viewport, NOT vertically centered. The bottom portion remains as empty space to fill the 100vh.

**Alignment:** left-aligned (not centered)

**Content width:** 720px max-width

**Layout (top to bottom, in order):**

1. **Terminal prompt** -- `TerminalPrompt` component from Phase 3. Renders `mattia@selfrules ~ $` in JetBrains Mono 14px, color #5A5A5E (text-tertiary), followed by amber block cursor (#E8A838). Margin-bottom: 24px to headline.

2. **Headline** -- Display size (Space Grotesk 700, clamp 48-64px), text-primary (#F5F5F0). The H1 of the page.

3. **Subtitle** -- Body size (Inter 400, 16px), line-height 1.7, text-secondary (#8A8A8E), 2-3 lines. At the END of the subtitle text, render a `BlinkingCursor` component with `type="underscore"` -- a blinking `_` character in accent color (#E8A838). Margin-top from headline: 16px.

4. **CTA buttons** -- Two buttons side by side, 16px gap between them. Margin-top from subtitle: 32px.

5. **Credential tags** -- Horizontal row of `Tag` components, 8px gap between tags. Margin-top from CTAs: 24px.

6. **Separator line** -- 1px height, ~200px width, color #1A1A1F (border-default). Margin-top from tags: 32px.

7. **Signature phrase** -- Italic text (Inter 400 italic, 14-16px, #8A8A8E). See Copywriting Contract for exact copy. Margin-top from separator: 16px.

8. **Empty space** -- Remaining viewport height. No content. Ensures hero fills 100vh.

### How I Work Section (HOME-02)
- Section label: label role (14px mono uppercase)
- Intro paragraph: body size (16px), line-height 1.7, text-secondary, max-width 720px
- Pillar grid: 3 columns at desktop (>=1024px), 1 column at mobile
- Each pillar: title size (Space Grotesk 700) for heading, body size for text, no background, no border
- Grid gap: 32px between columns, 48px between rows on mobile

### Journey Section (HOME-03)
- Horizontal timeline at desktop (>=1024px): 4 blocks in a row with visual connector line
- Vertical timeline at mobile: stacked blocks with vertical connector line
- Connector: 1px line in border-default color
- Each block: title size (Space Grotesk 700) for role, body for company/text, dates in label role (14px mono, text-secondary)
- Overlapping phases (Designer 2012-18 and Developer 2016-20): visually represented through overlapping date ranges
- Closing phrase: body size (16px), line-height 1.7, text-primary, 48px margin-top, max-width 720px

### Numbers Section (HOME-04)
- Grid: 3 columns x 2 rows at desktop (>=1024px), 2x3 at tablet (>=640px), 1 column at mobile
- Container: wide-width (1080px)
- Grid gap: 24px
- MetricCard numbers use `CountUpNumber` component: animated count-up from 0 to target value
  - Trigger: IntersectionObserver at threshold 0.3 (fires when 30% of Metrics section is visible)
  - Duration: 800ms per number
  - Easing: ease-out
  - Each number animates independently
  - `prefers-reduced-motion`: number appears immediately at final value, no animation
- MetricCard hover: border transitions from border-default to border-accent, 150ms ease
- Below grid: text link "Read the full case studies" with arrow, text-secondary, accent on hover

### Current Work Section (HOME-05)
- 2 cards side by side at desktop (>=768px), stacked at mobile
- Card: 1px border border-default, bg-surface background, 24px padding, 0px border-radius
- Card hover: border-accent + translate-y(-2px), 150ms transition
- Gap between cards: 24px

### Contact Section (HOME-06)
- Max-width: 720px
- Email: title size (Space Grotesk 700), JetBrains Mono, text-primary, transitions to accent on hover
- Links (LinkedIn, CV): body size, text-secondary, accent on hover, arrow suffix
- Availability tag: Tag component style (label role: mono, uppercase, text-secondary), inline display
- Vertical spacing between elements: 16px between text items, 32px between groups

## Interaction States

| Element | Default | Hover | Active | Focus |
|---------|---------|-------|--------|-------|
| Primary CTA button | bg-accent, text #0A0A0B | translate-y(-1px), bg-accent-hover | translate-y(0) | 2px outline accent, 2px offset |
| Secondary CTA button | bg-transparent, border border-default, text-primary | border-accent, text-accent | opacity 0.9 | 2px outline accent, 2px offset |
| MetricCard | border border-default | border-accent, 150ms ease | -- | 2px outline accent, 2px offset |
| "Now" card | border border-default, bg-surface | border-accent, translate-y(-2px), 150ms | -- | 2px outline accent, 2px offset |
| Email link | text-primary, font-mono | text-accent, 150ms | underline | 2px outline accent, 2px offset |
| Text link | text-secondary | text-accent, 150ms | underline | 2px outline accent, 2px offset |
| BlinkingCursor | opacity 1 | -- | -- | -- |
| CountUpNumber | 0 (pre-animation) | -- | -- | -- |

All transitions: 150ms ease timing function.
All focus styles: visible outline (not outline-none). WCAG 2.1 AA compliant focus indicators.

Animations:
- `BlinkingCursor`: CSS keyframes `opacity 0 <-> 1`, `step-end`, 1s infinite loop. Client component. Respects `prefers-reduced-motion` (cursor shown static, no blink).
- `CountUpNumber`: requestAnimationFrame-driven count from 0 to target, 800ms ease-out. Client component. Respects `prefers-reduced-motion` (number shown at final value immediately).

Source: figma-make-prompt.md micro-interactions, implementation-plan.md Button.tsx spec, DESIGN-UPDATE-v25.md Changes 3, 6

## Accessibility Contract

| Requirement | Specification |
|-------------|---------------|
| Semantic HTML | Hero uses `<section>` with aria-label, each section is a `<section>`, contact email is an `<a href="mailto:">` |
| Heading hierarchy | H1: hero headline (one per page), H2: section titles, H3: pillar/card titles |
| Skip to content | Provided by Phase 2 shell layout |
| Focus styles | All interactive elements have visible 2px accent outline with 2px offset |
| Color contrast | Text primary #F5F5F0 on #0A0A0B = 17.4:1 (AAA). Text secondary #8A8A8E on #0A0A0B = 4.9:1 (AA). Accent #E8A838 on #0A0A0B = 8.6:1 (AAA). Text tertiary #5A5A5E on #0A0A0B = 3.1:1 (decorative only -- terminal prompt is not essential content). |
| Reduced motion | `BlinkingCursor`: cursor shown static (no animation) when `prefers-reduced-motion: reduce`. `CountUpNumber`: number appears at final value immediately (no count-up animation) when `prefers-reduced-motion: reduce`. |
| Language | `lang` attribute set by Phase 1 i18n routing. Content matches declared language. |
| Link purpose | All links have descriptive text (no "click here"). CTA buttons use verb + noun pattern. |
| Decorative elements | Terminal prompt and blinking cursor are decorative/atmospheric. Use `aria-hidden="true"` on TerminalPrompt to prevent screen readers from announcing `mattia@selfrules ~ $`. |
