```typescript
  }>;

```

Layout (from UI-SPEC "How I Work Section"):
- Use `<Section>` wrapper (default 720px width for intro, but pillars need wider)
- `<SectionHeader label={label} title={title} />`
- Intro paragraph: `<p className="text-base leading-[1.7] text-secondary mb-12 max-w-[720px]">`
- Pillar grid: `<div className="grid grid-cols-1 lg:grid-cols-3 gap-8 lg:gap-8">` (3 cols at >=1024px, 1 col mobile, 32px gap)
- Each pillar: `<div>` with `<h3 className="text-[clamp(28px,3vw,36px)] font-bold leading-[1.2] text-primary mb-4">` for title (Title role), `<p className="text-base leading-[1.6] text-secondary">` for text (Body role)
- Mobile gap between pillars: 48px (`gap-12` via Tailwind, which is 48px)
- Correct: `gap-12 lg:gap-8` for 48px mobile, 32px desktop

For bold numbers in pillar text (e.g., **25%**): the page.tsx will use `t.rich()` to resolve `<b>` tags into `<strong className="font-bold text-accent">`. The pillar component receives the resolved ReactNode. Change `text` type to `React.ReactNode` to support this.
