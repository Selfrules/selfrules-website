## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, tag gaps |
| md | 16px | Default element spacing, card internal padding minimum |
| lg | 24px | Card padding, button padding vertical |
| xl | 32px | Sub-section breaks, grid gaps |
| 2xl | 48px | Intra-section spacing |
| 3xl | 64px | Minor section breaks |
| section-gap | clamp(80px, 10vw, 160px) | Vertical space between homepage sections |
| page-padding | clamp(20px, 5vw, 80px) | Horizontal page padding (responsive) |

Exceptions:
- Hero section uses `min-h-screen` (100vh) for full viewport height
- Button padding: 12px vertical, 24px horizontal (specified in implementation-plan.md)

Content width constraints (not spacing, but layout):
- `content-width`: 720px max-width for text blocks (Section wrapper default)
- `wide-width`: 1080px max-width for metrics grid and wider layouts (Section wrapper wide variant)

## Layout Contract

### Hero Section (HOME-01)
- Height: `min-h-screen`, content vertically centered or upper-third aligned
- Alignment: left-aligned (not centered)
- Content width: 720px max-width
- Headline: hero-headline size, text-primary
- Subtitle: body-large size, text-secondary, 2-3 lines
- CTAs: two buttons side by side, 16px gap between them
- Credential tags: horizontal row below CTAs, 24px margin-top from CTAs, 8px gap between tags

### How I Work Section (HOME-02)
- Section label: 14px mono uppercase above title
- Intro paragraph: body-large, text-secondary, max-width 720px
- Pillar grid: 3 columns at desktop (>=1024px), 1 column at mobile
- Each pillar: card-title for heading, body for text, no background, no border
- Grid gap: 32px between columns, 48px between rows on mobile

### Journey Section (HOME-03)
- Horizontal timeline at desktop (>=1024px): 4 blocks in a row with visual connector line
- Vertical timeline at mobile: stacked blocks with vertical connector line
- Connector: 1px line in border-default color
- Each block: card-title for role, body for company/text, dates in mono 14px text-secondary
- Overlapping phases (Designer 2012-18 and Developer 2016-20): visually represented through overlapping date ranges
- Closing phrase: body-large, text-primary, 48px margin-top, max-width 720px

### Numbers Section (HOME-04)
- Grid: 3 columns x 2 rows at desktop (>=1024px), 2x3 at tablet (>=640px), 1 column at mobile
- Container: wide-width (1080px)
- Grid gap: 24px
- MetricCard hover: border transitions from border-default to border-accent, 150ms ease
- Below grid: text link "Read the full case studies" with arrow, text-secondary, accent on hover

### Current Work Section (HOME-05)
- 2 cards side by side at desktop (>=768px), stacked at mobile
- Card: 1px border border-default, bg-surface background, 24px padding, 0px border-radius
- Card hover: border-accent + translate-y(-2px), 150ms transition
- Gap between cards: 24px

### Contact Section (HOME-06)
- Max-width: 720px
- Email: JetBrains Mono, 20-24px, text-primary, transitions to accent on hover
- Links (LinkedIn, CV): body size, text-secondary, accent on hover, arrow suffix
- Availability tag: Tag component style (mono, uppercase, text-secondary), inline display
- Vertical spacing between elements: 16px between text items, 32px between groups

## Accessibility Contract

| Requirement | Specification |
|-------------|---------------|
| Semantic HTML | Hero uses `<section>` with aria-label, each section is a `<section>`, contact email is an `<a href="mailto:">` |
| Heading hierarchy | H1: hero headline (one per page), H2: section titles, H3: pillar/card titles |
| Skip to content | Provided by Phase 2 shell layout |
| Focus styles | All interactive elements have visible 2px accent outline with 2px offset |
| Color contrast | Text primary #F5F5F0 on #0A0A0B = 17.4:1 (AAA). Text secondary #8A8A8E on #0A0A0B = 4.9:1 (AA). Accent #E8A838 on #0A0A0B = 8.6:1 (AAA). |
| Reduced motion | No animations in Phase 4 (animations added in Phase 6). No motion to disable. |
| Language | `lang` attribute set by Phase 1 i18n routing. Content matches declared language. |
| Link purpose | All links have descriptive text (no "click here"). CTA buttons use verb + noun pattern. |
