# Plan 06-04: Vercel Deploy and Domain Configuration

## Objective

Deploy the site to Vercel, configure the selfrules.org custom domain with HTTPS, and set up www-to-non-www redirect. After this plan, the site is live and publicly accessible at https://selfrules.org.

## Tasks

<task id="06-04-01">
<title>Verify build and connect to Vercel</title>
<read_first>
- package.json (verify build script)
- next.config.ts or next.config.mjs (verify no issues)
- vercel.json (if exists)
</read_first>
<action>
1. Run a final `npx next build` to verify clean build with no errors.
2. Check if the repo is already connected to a Vercel project:
   - Run `vercel ls` to list existing projects
   - If a project exists for this repo, use it
   - If not, run `vercel link` to connect the repo to a Vercel project

3. If `vercel.json` does not exist, no action needed -- Vercel auto-detects Next.js.
   If it exists, verify it doesn't contain conflicting settings.

4. Set any required environment variables:
   - `NEXT_PUBLIC_UMAMI_WEBSITE_ID` (if Umami is configured)
   - Run `vercel env add NEXT_PUBLIC_UMAMI_WEBSITE_ID` if needed

5. Deploy a preview build:
   ```bash
   vercel
   ```
   Verify the preview URL works:
   - All pages load correctly
   - i18n routing works (/ for EN, /it for IT)
   - No 500 errors
</action>
<acceptance_criteria>
- npx next build completes with exit code 0
- vercel preview deploy succeeds and returns a preview URL
- Preview URL is accessible and returns HTTP 200
- curl -s {preview-url} | grep '<title>' shows correct homepage title
- curl -s {preview-url}/it | grep '<title>' shows Italian homepage title
</acceptance_criteria>
</task>

<task id="06-04-03">
<title>Configure selfrules.org domain and HTTPS</title>
<read_first>
- Vercel project settings (from task 06-04-02)
</read_first>
<action>
1. Add the custom domain to the Vercel project:
   ```bash
   vercel domains add selfrules.org
   ```

2. Follow Vercel's DNS configuration instructions:
   - If using Vercel as DNS: update nameservers at domain registrar
   - If keeping external DNS: add A record (76.76.21.21) and/or CNAME record as instructed by Vercel

3. Vercel automatically provisions HTTPS via Let's Encrypt. Wait for SSL certificate to be issued.

4. Verify HTTPS is working:
   ```bash
   curl -I https://selfrules.org
   ```
   Should return HTTP/2 200 with valid SSL.

NOTE: This task may require Mattia's manual intervention to update DNS records at the domain registrar. If `vercel domains add` fails due to existing configuration, check if selfrules.org is already connected to another Vercel project (the old site). If so, the domain needs to be moved to the new project.

IMPORTANT: If the domain is already connected to an existing Vercel project, Mattia needs to either:
- Remove the domain from the old project first, then add to new project
- Or update the existing project to point to the new repo
</action>
<acceptance_criteria>
- vercel domains add selfrules.org succeeds (or domain is already configured)
- curl -I https://selfrules.org returns HTTP 200 with valid SSL
- https://selfrules.org loads the new site content (not old site)
</acceptance_criteria>
</task>

<task id="06-04-04">
<title>Configure www redirect</title>
<read_first>
- Vercel project settings
</read_first>
<action>
1. Add www.selfrules.org as a redirect domain:
   ```bash
   vercel domains add www.selfrules.org
   ```
   Vercel should automatically set up the redirect from www to non-www (or prompt for configuration).

2. If Vercel doesn't auto-redirect, configure in Vercel dashboard:
   - Go to Project Settings > Domains
   - Ensure www.selfrules.org redirects to selfrules.org (301 redirect)

3. Verify the redirect:
   ```bash
   curl -I https://www.selfrules.org
   ```
   Should return 301 or 308 redirect to https://selfrules.org

4. Also verify HTTP redirects to HTTPS:
   ```bash
   curl -I http://selfrules.org
   ```
   Should return 301/308 redirect to https://selfrules.org
</action>
<acceptance_criteria>
- curl -I https://www.selfrules.org returns 301 or 308 status with Location header pointing to https://selfrules.org
- curl -I http://selfrules.org returns redirect to https://selfrules.org
- The final destination https://selfrules.org returns HTTP 200
</acceptance_criteria>
</task>
