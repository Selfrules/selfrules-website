1. Create `src/app/[locale]/[...rest]/page.tsx` -- the catch-all route that triggers localized 404:
```typescript
import { notFound } from 'next/navigation';

export default function CatchAllPage() {
  notFound();
}
```

This file MUST exist for localized 404 to work. Without it, Next.js falls back to the root `app/not-found.tsx` which has no locale context, no shell layout, no translations. The catch-all intercepts any unknown route within the `[locale]` segment and calls `notFound()`, which renders `[locale]/not-found.tsx` within the locale layout (navbar + footer).

2. Create `src/app/[locale]/not-found.tsx`:
```typescript
import { getTranslations } from 'next-intl/server';
import { Link } from '@/i18n/navigation';

export default async function NotFoundPage() {
  const t = await getTranslations('notFound');

  return (
    <div className="flex min-h-[calc(100vh-64px)] flex-col items-center justify-center px-[--spacing-page-padding]">
      {/* Headline per D-12, D-13. Display size per UI-SPEC typography. */}
      <h1 className="text-[length:--text-display] font-bold leading-[1.1] text-text-primary text-center max-w-[720px]">
        {t('headline')}
      </h1>

      {/* Back link per D-12. Arrow prefix, accent color per UI-SPEC. */}
      <Link
        href="/"
        className="mt-8 text-[length:--text-body] text-accent transition-colors duration-150 hover:text-accent-hover"
      >
        &rarr; {t('backLink')}
      </Link>
    </div>
  );
}
```

Key decisions per locked decisions:
- D-12: Full shell (navbar + footer are provided by the locale layout that wraps this page). Content centered vertically and horizontally. No illustrations or decorative elements.
- D-13: Copy from i18n messages (COPY-04 compliance). EN: "This page doesn't exist. But the rest of the site does." IT: "Questa pagina non esiste. Ma il resto del sito si."
- UI-SPEC: Display size `clamp(40px, 5vw, 64px)` via `--text-display` token. Font weight 700. Line height 1.1.
- `min-h-[calc(100vh-64px)]` accounts for the 64px navbar offset on main (from Plan 01 layout).
- Server Component -- uses `getTranslations` (server-side), NOT `useTranslations`. No `'use client'` directive.
- `max-w-[720px]` on the headline to prevent super-wide text on large screens.
- Arrow prefix followed by the back link text.
- Link uses next-intl `Link` component (from `@/i18n/navigation`) so locale prefix is handled automatically.

NOTE: Do NOT add `setRequestLocale()` in not-found.tsx -- it's a special error boundary file. The locale context comes from the parent layout. Per next-intl docs, `setRequestLocale` should NOT be called in not-found.tsx because the locale is inherited from the layout.
