```typescript
  variant: 'primary' | 'secondary';
  children: React.ReactNode;

  className?: string;
}
```

**Implementation rules (per UICM-01 and UI-SPEC interaction contract):**

Base classes (both variants):
`inline-flex items-center justify-center h-12 px-8 text-base font-normal transition-all duration-150`

Primary variant classes:
`bg-accent text-dark hover:bg-accent-hover hover:-translate-y-[1px]`

Secondary variant classes:
`bg-transparent border border-default text-primary hover:border-accent hover:text-accent`

Focus styles (per A11Y-02):
`focus-visible:outline-2 focus-visible:outline-accent focus-visible:outline-offset-2`

**Polymorphic rendering (Claude's discretion -- single component chosen per research recommendation):**
- If `href` is provided: render as `<Link href={href} className={classes}>{children}</Link>`
- If `href` is absent: render as `<button type="button" className={classes} {...rest}>{children}</button>`
- Spread remaining props onto the rendered element (but type the rest appropriately)

**Key details:**
- Arrow suffix (" ->") is part of label text passed via children, NOT generated by the component
- Height is h-12 (48px), horizontal padding is px-8 (32px)
- font-normal (weight 400) for button text
- 0px border-radius (global rule, no rounded-* classes)
- All hover states are CSS-only -- no JavaScript, no 'use client'
- next/link can be imported in Server Components without 'use client'

**IMPORTANT:** The CSS variable names MUST match what Phase 1 defined in globals.css @theme. Read globals.css first and use the exact variable names.
