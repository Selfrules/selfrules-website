Implementation (per UICM-02, D-03, UI-SPEC):
```typescript
export function MetricCard({ number, label, context, className }: MetricCardProps) {
  return (
    <div
      className={cn(
        'bg-primary p-8',
        'outline outline-0 hover:outline-1 hover:outline-accent/40 hover:-outline-offset-1',
        'transition-all duration-200',
        className
      )}
    >
      <p className="font-mono font-bold text-accent text-[clamp(40px,5vw,72px)] leading-none">
        {number}
      </p>
      <p className="text-[13px] text-primary mt-4">{label}</p>
      <p className="text-[13px] text-tertiary mt-1">{context}</p>
    </div>
  );
}
```

Key details:
- MetricCard itself is a CELL inside a grid. The grid container (gap-[1px] bg-[color] border pattern) is built by the PAGE, not this component. Per D-03.
- Hover uses outline approach (not border) because the gap-[1px] grid means cells have no own border. outline-0 default, hover:outline-1 with accent/40 color.
- Number uses `text-[clamp(40px,5vw,72px)]` -- NO spaces in clamp() (Tailwind v4 pitfall). If Tailwind v4 does not support accent/40 on outline, use `hover:outline-[rgba(232,168,56,0.4)]` instead.
- label is text-primary (13px), context is text-tertiary (13px) -- tertiary is acceptable here per UI-SPEC WCAG note (supplementary text).
